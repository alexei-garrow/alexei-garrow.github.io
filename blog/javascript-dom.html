Control flow

What is control flow? Control flow is a term used to describe the way the programming language is run. In this case, by default JavaScrip ( and most other languages ) run from top to bottom, not unlike when reading a book. Control flow is hinted to in it's name, to control the flow. When we think of the standard top to bottom control flow though, this isn't always ideal. Sometimes we need to do multiple things before going to the next thing, just like in real life!

(see below)

Loops in control flow

So what is a loop, and how can we demonstrate control flow with it? Well firstly we need to get something to run through this loop. In this example below I'm going to create an array of ingredients. (I'll explain how that works below)

Here we declare the type of variable it is with the word 'let'. There are 2 main types, let and const (constant). We use const for variables we don't plan on chaning ( we can change things in the array though!) and we use let for a variable we might want to change later.

so let and const are the decleration, in this case 'ingredientsList' is the variable ( container for holding things like data). In this container we will store a type of data called an array. Arrays are indexs of data, or arrays of data stored as an index, whatever makes more sense to you. To tell it's an array we can see the [] square brackets.

Now that we have our array we need to store some information in it. In this case we need to store a list of ingredients! The way we do this is with the '' we put the word inside like 'food' then if we want more than one ( hint we should or why use an array!?), we can use a comma , to divide them like so: ['pizza', 'noodles', 'pasta']. We then end the line with a simicolon ;

Now that we have our array, we need to loop through it to take all the ingredients out and or check they are there!

Ok so what is going on here! First 'for' is the type of loops we then tell it what to do in the () brackets. firstly we say 'let i;'' this is assigning 'i' as the iterator we then tell it to start from 0 time through the loop. We are then giving it a condition, in this case as long as 'i' is less than the ingredientsList.length ( the length of our array) it will keep going. We then make sure to increase the number our iterator is searching through each time it collects data. We do this by assigning 'i' the '++' (increment assignment). This increases the iterator, therefor increasing the number of the index it is searching through.

The iterator starts at the first item of our array ' 0 ' yes 0 it starts 0 1 2 3 etc. each time it ++ it goes to the next so 0 loop back then 1 loop back etc, then finishes at the end. Thankfully we assigned the i less than ingredientsList.length, the main advantage of that is not only, not having to remember how long the array is, but also being able to add or remove from it.

We can do this with push and pop commands.

The DOM

The DOM or document object model, is simply a document that your browser creates. Everything inside this document is an object / node. JavaScript uses the DOM and it's objects etc, to interract with the elements of the page. JavaScript is run in the browser, and this is how it's done. JavaScript is brought into the HTML doc through us linking our .js file, it them uses the DOM to execute the code.

If we were to use an example of how or why we'd use the DOM, there are many, from simply changing the style of an element to having an interractive button and or form. JavaScrip can technically change anything on your website.

Functions and why we use them

Functions are reuseable sections of code. THey cna be very powerful for basic things like adding numbers together. The main benefit to this is in parameters like function addition(num1,num2) { return num1 + num 2; } console.log(addition(1,1)); this will add 1 + 1. The reason this is power is we can do this mutiple times throughout the code, we can call in that same function with a different arguement console.log(addition(200,50)); Without having to rewrite everything!
